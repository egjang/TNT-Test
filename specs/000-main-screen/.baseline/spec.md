# Feature Specification: Main Screen Layout

**Feature Branch**: `[000-main-screen]`
**Created**: [DATE]
**Status**: Initial Implementation (v0)
**Input**: User description: "메인 화면을 세로로 3등분(좌: 메뉴, 중: 업무, 우: 예약), 각 영역은 마우스로 크기 조정, 첫 메뉴는 수요관리. 향후 메인 화면 구성 기능만 추가 제공."

## Overview
- Goal: 기본 작업 동선의 기반이 되는 3컬럼 메인 레이아웃 제공
- Primary users: 내부 운영/업무 담당자
- Success criteria: 레이아웃이 직관적이고, 좌/우 폭을 유연하게 조절하며(최소폭 보장), 중앙 업무 영역을 안정적으로 확보

## User Scenarios
- [ ] 사용자는 앱에 접속하면 좌/중/우 3개 영역으로 분할된 메인 화면을 본다
- [ ] 사용자는 좌/우 경계선을 드래그하여 패널 너비를 조정할 수 있다(최소폭 이하로 축소 불가)
- [ ] 사용자는 좌측 메뉴에서 "수요관리"를 선택할 수 있다(선택 상태 표시)
- [ ] 사용자는 중앙 영역에서 선택된 메뉴(초기: 수요관리)의 플레이스홀더 콘텐츠를 본다
- [ ] 사용자는 우측 영역이 향후 기능을 위해 예약되어 있음을 확인한다

## Functional Requirements
- [ ] FR-1: 메인 화면은 세로 방향 3분할(좌/중/우) 레이아웃을 제공한다
- [ ] FR-2: 좌/우 패널 경계선 드래그로 너비를 조정할 수 있다(최소폭 보장, 중앙 영역은 가변)
- [ ] FR-3: 좌측 패널은 메뉴 영역이며, 첫 번째 항목으로 "수요관리"를 제공한다
- [ ] FR-4: 중앙 패널은 업무 메인 영역이며, 현재 선택된 메뉴의 내용을 표시한다(초기 버전은 수요관리 플레이스홀더)
- [ ] FR-5: 우측 패널은 향후 기능을 위한 예약 영역으로 표시한다
- [ ] FR-6: 좌/우 패널 너비는 재접속 시에도 유지되도록 저장한다(브라우저 로컬 저장소)

## Out of Scope
- 라우팅/URL 동기화, 인증/권한, 실제 수요관리 업무 기능 구현, 우측 패널 상세 기능, 반응형 모바일 최적화

## Risks & Constraints
- 최소폭 미준수 시 중앙 영역과 겹침 위험 → 드래그 로직에서 총 폭 대비 최소 여백 확보
- 다양한 해상도에서 레이아웃 일관성 유지 필요

## Acceptance Criteria
- [ ] AC-1: 화면에 좌/중/우 3개 영역이 보이고, 좌측에 "메뉴", 중앙에 "업무", 우측에 "우측 패널" 헤더가 표시된다
- [ ] AC-2: 좌/우 경계선을 드래그하면 해당 패널 너비가 변경되고 중앙 영역이 자동으로 재배치된다(최소폭 이하로 축소되지 않음)
- [ ] AC-3: 좌측 메뉴에 "수요관리"가 보이고 클릭 시 활성화 표시가 된다
- [ ] AC-4: 중앙 영역에는 "수요관리" 플레이스홀더 텍스트가 표시된다
- [ ] AC-5: 새로고침 후에도 최근 좌/우 패널 너비가 유지된다
